# AI漫剧协同制作平台 - 界面原型说明文档

## 一、项目概述

### 1.1 项目背景
本原型系统基于《AI漫剧协同制作平台功能需求框架》开发，实现了AI制作工具模块和团队协作与流程管理模块的核心功能界面。

### 1.2 技术栈
- **前端框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI组件库**: Element Plus
- **状态管理**: Pinia
- **样式方案**: Tailwind CSS
- **路由管理**: Vue Router 4

## 二、核心功能模块

### 2.1 AI制作工具模块

#### 2.1.1 剧本编辑器 (ScriptEditorView)
**功能描述**: 提供剧本文本编辑、一键解析、版本管理等功能。

**主要特性**:
- 富文本编辑器支持Markdown格式
- 实时字数、行数统计
- 剧本大纲侧边栏
- 一键AI解析按钮
- 版本历史查看
- 剧本锁定功能

**页面路径**: `/script`

#### 2.1.2 剧本解析结果 (ScriptParseView)
**功能描述**: 展示AI解析剧本后的结构化结果，支持人工审核和修正。

**主要特性**:
- 世界观描述展示与编辑
- 角色列表（支持增删改）
- 场景列表（支持增删改）
- 物品列表（支持增删改）
- 剧情节奏可视化
- 确认保存到资产库

**页面路径**: `/script/parse`

#### 2.1.3 资产库 (AssetListView)
**功能描述**: 统一管理角色、场景、物品等资产。

**主要特性**:
- 资产类型筛选（全部/角色/场景/物品）
- 网格布局展示
- 资产搜索功能
- 快速新建资产
- 资产卡片悬停操作

**页面路径**: `/assets`

#### 2.1.4 资产详情 (AssetDetailView)
**功能描述**: 展示单个资产的详细信息，支持图片生成和管理。

**主要特性**:
- 多版本图片切换
- 提示词编辑器
- AI优化提示词
- 引用追踪（显示被哪些镜头使用）
- 生成历史
- 下载/分享操作

**页面路径**: `/assets/:id`

#### 2.1.5 分镜表 (StoryboardView)
**功能描述**: 表格形式管理所有镜头，支持Excel导入导出。

**主要特性**:
- 镜头列表表格（编号、描述、台词、景别、角度、运镜、时长）
- 镜头预览缩略图
- 状态标签显示
- 添加/编辑/删除镜头
- Excel导入/导出
- 批量生成按钮

**页面路径**: `/storyboard`

#### 2.1.6 镜头详情 (ShotDetailView)
**功能描述**: 单个镜头的详细编辑页面，支持图片生成和审核。

**主要特性**:
- 图片预览区域
- 镜头信息编辑（编号、描述、台词、景别、角度、运镜、时长）
- 提示词编辑器
- 绑定资产（角色/场景）
- 通过/驳回审核按钮
- 重新生成功能

**页面路径**: `/shots/:id`

#### 2.1.7 生成中心 (GenerationView)
**功能描述**: 管理AI图片/视频生成任务队列。

**主要特性**:
- 任务统计卡片（排队中、运行中、成功率、总成本）
- 任务列表（状态、进度条、预览、操作）
- 状态筛选（全部/排队/运行中/已完成）
- 失败重试功能
- 模型性能统计

**页面路径**: `/generation`

## 三、设计思路

### 3.1 整体布局
采用经典的侧边栏+顶部导航+内容区的三栏布局：
- **左侧边栏**: 项目切换、主导航菜单、用户信息
- **顶部导航**: 面包屑、全局搜索、通知、帮助
- **内容区**: 页面主体内容

### 3.2 视觉风格
- **主色调**: 蓝色系（#0ea5e9）作为品牌色
- **背景色**: 浅灰色（#f8fafc）营造舒适感
- **卡片设计**: 白色背景、圆角、轻微阴影
- **字体**: Inter字体家族，保证可读性

### 3.3 交互设计
- **悬停效果**: 卡片阴影加深、按钮颜色变化
- **过渡动画**: 页面切换、弹窗打开关闭的平滑过渡
- **反馈机制**: 操作成功/失败的Toast提示
- **加载状态**: 骨架屏和加载动画

## 四、数据模型

### 4.1 核心实体关系
```
Project (项目)
├── Script (剧本)
│   └── ScriptParseResult (解析结果)
│       ├── Character[] (角色)
│       ├── Scene[] (场景)
│       └── Item[] (物品)
├── Entity (资产)
│   ├── Character (角色)
│   ├── Scene (场景)
│   └── Item (物品)
├── Storyboard (分镜表)
│   └── Shot[] (镜头)
├── Task (任务)
└── Comment (评论)
```

### 4.2 状态管理
使用Pinia进行全局状态管理，按模块划分：
- `projectStore`: 项目相关状态
- `scriptStore`: 剧本相关状态
- `assetStore`: 资产相关状态
- `storyboardStore`: 分镜相关状态
- `taskStore`: 任务相关状态
- `generationStore`: 生成任务相关状态
