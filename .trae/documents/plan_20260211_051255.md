# UI 一致性修复与优化计划

根据您的要求，我制定了以下修复计划，以解决剧本编辑器和分镜编辑界面的 UI 一致性问题。

## 1. 剧本编辑器界面 (script/edit/{id}) AI 工具模块改造

**文件**: `src/views/script/ScriptEditorView.vue`

### 目标
将 AI 模型选择下拉框从右侧侧边栏的 AI 工具卡片中移除，并将其重新定位到页面顶部的标题栏区域（左侧靠近版本历史按钮的位置）。

### 实施步骤
1.  **移动组件**: 将 `<el-select>` 组件从 `<!-- AI Tools -->` 卡片区域移动到 `<!-- Page Header -->` 的右侧操作区域。
2.  **样式调整**:
    *   移除 `w-full` 类，改为固定宽度（如 `w-40`）或根据内容自适应。
    *   确保下拉框高度、边框颜色、字体与 Header 中的按钮保持视觉协调（使用 `size="default"` 或调整 class）。
    *   保留 `v-model="selectedAiModel"`, `@change`, `:loading` 等原有逻辑。
3.  **布局优化**: 确保 Header 右侧按钮组（版本历史、上传、一键解析等）与新加入的 Select 之间有适当间距（`gap`）。
4.  **清理代码**: 删除原 AI 工具卡片中不再需要的 `model-selector` div 容器。

## 2. 分镜编辑界面 (shots/shot{id}) 视频生成模块优化

**文件**: `src/views/storyboard/ShotDetailView.vue`

### 目标
统一视频生成模块与图片生成模块中模型选择下拉框的 UI 样式，移除不一致的高亮效果。

### 实施步骤
1.  **样式统一**:
    *   检查图片生成模块 (`Image Generation Module`) 的 `<el-select>` 配置（如 `size="small"`, `class="w-40"`）。
    *   将视频生成模块 (`Video Generation Module`) 的 `<el-select>` 调整为完全一致的配置。
    *   移除视频下拉选项中特殊的背景色逻辑（如 `:class="{ 'bg-yellow-50': model.recommended }"`），使其与图片模块的简洁风格保持一致，或仅保留标签 (`el-tag`) 提示。
    *   确保 Tooltip 逻辑（如果图片模块有）在视频模块也得到适当处理（根据需求，图片模块有 Tooltip，视频模块目前没有明确要求加 Tooltip，但要求样式一致，重点是下拉框本身的外观）。
2.  **代码复用**: 确认两个 Select 的 `placeholder`、`size`、`class` 等属性值一致。

## 3. 验证与测试
*   **功能测试**: 确认剧本页顶部下拉框切换模型后，`handleModelChange` 仍正常触发且 Toast 提示正确。
*   **UI 走查**:
    *   对比剧本页 Header 的新布局，确保不拥挤、不换行。
    *   对比分镜页图片和视频两个模块的下拉框，确保视觉上（高度、边框、字体、下拉菜单样式）完全一致。

我将按照此计划执行代码修改。
